# i18n — SK / EN / DE

Geo-lokalizácia viditeľných UI textov na `growmedica.cz` bez zmeny URL slugov.

## Jazyky

| Kód | Použitie |
|-----|----------|
| `sk` | Slovensko, Česko |
| `de` | Nemecko, Rakúsko, Švajčiarsko + default |
| `en` | Ostatné krajiny |

## Detekcia (priorita)

1. `?lang=sk|en|de` — nastaví cookie a redirect na čistú URL
2. Cookie `growmedica_locale` (30 dní)
3. Header `x-vercel-ip-country` (Vercel geo)
4. `Accept-Language`
5. `NEXT_PUBLIC_DEFAULT_LOCALE` (default: `sk` pre growmedica.cz)

## Env

```bash
NEXT_PUBLIC_SITE_URL=https://growmedica.cz
NEXT_PUBLIC_DEFAULT_LOCALE=sk
```

## Architektúra

| Súbor | Úloha |
|-------|--------|
| `src/lib/i18n/locales/*.json` | Prekladové reťazce |
| `src/lib/i18n/detect.ts` | `resolveLocaleFromInput()` |
| `src/lib/i18n/translate.ts` | `t(key, locale)` |
| `src/middleware.ts` | Geo/cookie detekcia, header `x-growmedica-locale` |
| `src/components/i18n/LocaleProvider.tsx` | Client context `useT()` |
| `src/components/i18n/LanguageSwitcher.tsx` | Prepínač SK/EN/DE v headeri |

## SEO — hreflang

`src/lib/seo.ts` → `buildHreflangLinks(pathname)` generuje URL s `?lang=sk|en|de`.

Hreflang `<link>` tagy sa renderujú cez `HreflangLinks` v layout (Next.js Metadata API stripuje `?lang=` na root path).

```
https://growmedica.cz?lang=sk
https://growmedica.cz/produkty?lang=en
```

Canonical URL je vždy rovnaká cesta bez `?lang=` (pre `/` bez trailing slash).

## Čo sa neprekladá

- WooCommerce produktové názvy a popisy
- URL slugy (`/produkty`, `/kolekcie`)
- JSON-LD structured data
- AI compliance pravidlá
- Dashboard agent (`/dashboard` zostáva SK)

## Nový prekladový kľúč

1. Pridaj kľúč do `sk.json`, `en.json`, `de.json`
2. Použi `t('namespace.key', locale)` v server komponente alebo `useT()` v client komponente
