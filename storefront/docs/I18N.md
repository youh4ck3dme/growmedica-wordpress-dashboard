# i18n — CS / SK / EN / DE

Geo-lokalizácia viditeľných UI textov na `growmedica.cz` bez zmeny URL slugov.

## Jazyky

| Kód | Použitie |
|-----|----------|
| `cs` | Česko (default) |
| `sk` | Slovensko |
| `de` | Nemecko, Rakúsko, Švajčiarsko |
| `en` | Ostatné krajiny |

## Detekcia (priorita)

1. `?lang=cs|sk|en|de` — nastaví cookie a redirect na čistú URL
2. Cookie `growmedica_locale` (30 dní)
3. Header `x-vercel-ip-country` (Vercel geo): CZ→`cs`, SK→`sk`, DE/AT/CH→`de`, inak `en`
4. `Accept-Language` (`cs` pred `sk`)
5. `NEXT_PUBLIC_DEFAULT_LOCALE` (default: `cs` pre growmedica.cz)

## Env

```bash
NEXT_PUBLIC_SITE_URL=https://growmedica.cz
NEXT_PUBLIC_DEFAULT_LOCALE=cs
```

## Architektúra

| Súbor | Úloha |
|-------|--------|
| `src/lib/i18n/locales/*.json` | Prekladové reťazce (`cs`, `sk`, `en`, `de`) |
| `src/lib/i18n/detect.ts` | `resolveLocaleFromInput()` |
| `src/lib/i18n/translate.ts` | `t(key, locale)` |
| `src/middleware.ts` | Geo/cookie detekcia, header `x-growmedica-locale` |
| `src/components/i18n/LocaleProvider.tsx` | Client context `useT()` |
| `src/components/i18n/LanguageSwitcher.tsx` | Compact dropdown: viditeľný len aktuálny locale; po kliknutí CS / SK / EN / DE |

## SEO — hreflang

`src/lib/seo.ts` → `buildHreflangLinks(pathname)` generuje URL s `?lang=cs|sk|en|de`.

`x-default` ukazuje na default locale (`cs`).

```
https://growmedica.cz?lang=cs
https://growmedica.cz/produkty?lang=sk
```

Canonical URL je vždy rovnaká cesta bez `?lang=` (pre `/` bez trailing slash).

## Čo sa neprekladá

- WooCommerce / Shopify produktové názvy a popisy
- URL slugy (`/produkty`, `/kolekcie`)
- JSON-LD structured data
- AI compliance pravidlá

## Nový prekladový kľúč

1. Pridaj kľúč do `cs.json`, `sk.json`, `en.json`, `de.json`
2. Použi `t('namespace.key', locale)` v server komponente alebo `useT()` v client komponente
